HTTP & Backend Foundations

You have consumed REST APIs for 14 years. Now you build them.

1HTTP — the language of the web

HTTP is the protocol behind every API call your mobile app makes. A client sends a request; a server returns a response. You've always been the client — now you become the server.

Request anatomy
Method: what to do (GET, POST…)
URL: where to do it
Headers: metadata (auth, content type)
Body: data payload (POST/PUT only)
Response anatomy
Status code: 200 OK, 404 Not Found…
Headers: Content-Type, Cache-Control…
Body: JSON, HTML, or file data
// fetch() — you know this side already
fetch('https://api.example.com/users/42', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json',
             'Authorization': 'Bearer <token>' },
  body: JSON.stringify({ name: 'Gokul' })
});

HTTP methods

Status codes

200 OK             // success, here's your data
201 Created        // new resource made
204 No Content     // success, nothing to return (DELETE)
400 Bad Request    // malformed request body / missing fields
401 Unauthorized   // not logged in — no token
403 Forbidden      // logged in but no permission
404 Not Found      // resource doesn't exist
422 Unprocessable  // validation failed
500 Server Error   // backend bug / crash

2Node.js — JavaScript on the server

Node.js is the V8 JavaScript engine extracted from Chrome and run directly on a machine — no browser. The same JS you write in React Native runs here, but with access to the file system, network sockets, and OS APIs instead of native UI components.

React Native runtime
V8/Hermes engine
Access to native APIs
Renders to UIKit / Android Views
Bundled as .ipa / .apk
Node.js runtime
V8 engine
Access to fs, net, http, os
Renders nothing — serves data
Runs as a long-lived process
// hello.js — run with: node hello.js
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({ message: 'Hello from Node!' }));
});

server.listen(3000, () => console.log('Listening on :3000'));

3Express.js — building REST APIs

Express is the minimal Node.js web framework. It adds routing, middleware, and request/response helpers on top of Node's raw HTTP module. Think of it as the bare-bones foundation most Node backends are built on.

const express = require('express');
const app = express();
app.use(express.json()); // parse JSON request bodies

// In-memory "database" for learning
let users = [{ id: 1, name: 'Gokul' }];

app.get('/users', (req, res) => {
  res.json(users);
});

app.get('/users/:id', (req, res) => {
  const user = users.find(u => u.id === +req.params.id);
  if (!user) return res.status(404).json({ error: 'Not found' });
  res.json(user);
});

app.post('/users', (req, res) => {
  const user = { id: Date.now(), ...req.body };
  users.push(user);
  res.status(201).json(user);
});

app.patch('/users/:id', (req, res) => {
  const user = users.find(u => u.id === +req.params.id);
  Object.assign(user, req.body);
  res.json(user);
});

app.delete('/users/:id', (req, res) => {
  users = users.filter(u => u.id !== +req.params.id);
  res.status(204).send();
});

app.listen(3000);

4Middleware — the Express pipeline

Middleware functions run between receiving a request and sending a response. They can log, authenticate, validate, transform — and either continue the chain or short-circuit with a response.

Mobile analogy: Middleware is like URLSession interceptors or Alamofire adapters — they sit between your code and the request/response, transforming or blocking as needed.
// Custom middleware function signature: (req, res, next)
const logger = (req, res, next) => {
  console.log(req.method, req.url, Date.now());
  next(); // continue to next middleware or route handler
};

const authGuard = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token' });
  next();
};

app.use(logger);                        // global — all routes
app.get('/protected', authGuard, handler); // route-level

5JWT authentication

JWT (JSON Web Token) is the standard for stateless API authentication. The server issues a signed token on login; the client sends it with every subsequent request in the Authorization header.

const jwt = require('jsonwebtoken');
const SECRET = process.env.JWT_SECRET;

// Login — issue a token
app.post('/login', async (req, res) => {
  const user = await findUserByEmail(req.body.email);
  if (!user || !await bcrypt.compare(req.body.password, user.hash))
    return res.status(401).json({ error: 'Invalid credentials' });
  const token = jwt.sign({ userId: user.id }, SECRET, { expiresIn: '7d' });
  res.json({ token });
});

// Auth middleware — verify token on protected routes
const authenticate = (req, res, next) => {
  try {
    const token = req.headers.authorization.split(' ')[1];
    req.user = jwt.verify(token, SECRET);
    next();
  } catch { res.status(401).json({ error: 'Invalid token' }); }
};
