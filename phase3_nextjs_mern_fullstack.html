Full Stack: Next.js & MERN

React Native to React web is a small step. Full-stack is one more.

1React for web — your RN knowledge transfers

React Native and React share the same core: components, props, state, hooks, context, and the reconciler. What changes is only the rendering target — HTML elements instead of native views.

React Native primitives
<View> <Text> <Image>
<TouchableOpacity>
<FlatList> <ScrollView>
StyleSheet.create({})
React web equivalents
<div> <p><span> <img>
<button> <a>
<ul><li> or Array.map
CSS / Tailwind classes
The hook layer is identical. useState, useEffect, useContext, useCallback, custom hooks — all behave exactly the same. Your 14 years of React Native hook knowledge transfers 100%.

2Next.js — Expo for the web

Plain React handles only UI. Next.js wraps React and adds routing, server-side rendering, API routes, image optimization, and deployment infrastructure. The same relationship as React Native → Expo.

File-based routing — the filesystem is your router

app/
├── page.tsx           →  https://myapp.com/
├── about/page.tsx     →  https://myapp.com/about
├── users/
│   ├── page.tsx       →  https://myapp.com/users
│   └── [id]/page.tsx  →  https://myapp.com/users/42   (dynamic)
└── api/
    └── users/route.ts →  https://myapp.com/api/users  (API endpoint)
Mobile analogy: Exactly like Expo Router — the file path is the route. You already know this mental model from your React Native work.

The 3 rendering modes — the key architectural concept

CSR
Client-Side Rendering
Empty HTML shell sent. React runs in the browser, fetches data after load.

Use for dashboards, anything behind auth.
SSG
Static Generation
HTML built at build time. Blazing fast, CDN-cacheable.

Use for blogs, docs, marketing pages.
SSR
Server-Side Rendering
HTML built per-request on the server. Fresh data + SEO.

Use for product pages, social feeds.
// SSR — async Server Component (Next.js 13+ App Router)
// This function runs on the SERVER, not in the browser
export default async function UserPage({ params }) {
  const user = await prisma.user.findUnique({ where: { id: +params.id } });
  return <div><h1>{user.name}</h1></div>;
}

// Force re-render on every request (opt into SSR)
export const dynamic = 'force-dynamic';

API Routes — backend in the same project

// app/api/users/route.ts
import { NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';

export async function GET() {
  const users = await prisma.user.findMany();
  return NextResponse.json(users);
}

export async function POST(req) {
  const body = await req.json();
  const user = await prisma.user.create({ data: body });
  return NextResponse.json(user, { status: 201 });
}

3The MERN stack

MongoDB + Express + React + Node. One language (JS/TS) from database to browser. The canonical choice when you want full-stack JavaScript without Next.js opinionation.

LetterTechnologyRole
MMongoDBDatabase — stores data as JSON documents
EExpressBackend — HTTP routing, middleware, API
RReactFrontend UI — component tree in the browser
NNode.jsRuntime — JS on the server
// Minimal MERN server (Express + Mongoose)
const express  = require('express');
const mongoose = require('mongoose');
const cors     = require('cors');

const app = express();
app.use(cors()); app.use(express.json());
mongoose.connect(process.env.MONGODB_URI);

const UserSchema = new mongoose.Schema({
  name:  String,
  email: { type: String, unique: true }
});
const User = mongoose.model('User', UserSchema);

app.get('/api/users',     async (_, res) => res.json(await User.find()));
app.post('/api/users',    async (req, res) => res.status(201).json(await User.create(req.body)));
app.listen(4000);

4Environment variables

# .env.local — NEVER commit to git, add to .gitignore
DATABASE_URL=postgresql://user:pass@localhost:5432/myapp
MONGODB_URI=mongodb://localhost:27017/myapp
JWT_SECRET=long-random-secret-here
NEXT_PUBLIC_API_URL=https://api.myapp.com   # browser-accessible
NEXT_PUBLIC_* variables are bundled into the client JS and visible to anyone in the browser. Never put secrets, API keys, or DB credentials in public variables.

5Deployment

Vercel (Next.js)
Push to GitHub → auto-deploys. Free tier handles SSR, CDN, edge functions. The default choice for Next.js projects. npx vercel to deploy.
Render (MERN backend)
Deploy Express/Node servers. Connect GitHub → auto-deploys on push. Free tier available. MongoDB via MongoDB Atlas (cloud-hosted).
The complete free-tier stack: React frontend on Vercel → calls Express API on Render → reads/writes MongoDB Atlas. Three services, zero cost, your first full-stack app is live.
