⚡ The Edge

Phase 5 put your app in one region. The Edge is about making it feel instant everywhere — pushing work as close to the user as possible and never doing the same work twice.

1Latency is a tax you pay by distance

Data can't beat the speed of light. A round trip across the planet costs ~150ms before your code runs. The fix: serve from a location near the user.

JavaScript · runnable
// Rough round-trip latency by distance (fibre ≈ 200,000 km/s)
const rtt = km => ((km * 2) / 200000 * 1000).toFixed(0) + ' ms';
console.log('same city   (~20km):  ', rtt(20));
console.log('same region (~500km): ', rtt(500));
console.log('cross-planet(~15000km):', rtt(15000));
console.log('→ a CDN edge near the user turns the bottom row into the top');

2The caching hierarchy

Every layer that answers without hitting your origin is a win. Check them in order; the earliest hit is the cheapest.

LayerLivesCaches
Browser cacheUser's deviceAssets, API responses
CDN edge~Near the userStatic files, cacheable pages
App cache (Redis)Your regionComputed/hot data (Phase 6)
DatabaseOriginSource of truth — last resort

Run the cache-aside pattern — first call is a slow miss, the rest are instant hits:

JavaScript · runnable
// Cache-aside: check cache → on miss, compute + store
const cache = new Map();
let dbHits = 0;
function getProfile(id) {
  if (cache.has(id)) return '⚡ HIT  ' + cache.get(id);
  dbHits++;
  const value = 'profile#' + id;        // "expensive" DB read
  cache.set(id, value);
  return '🐢 MISS ' + value;
}
[7, 7, 7, 9].forEach(id => console.log(getProfile(id)));
console.log('DB was hit only', dbHits, 'times for 4 requests');
"There are only two hard things…" — cache invalidation is one of them. Use short TTLs, or bust the key on write. Stale data is the price of speed; choose the TTL deliberately.

3Core Web Vitals — measure what users feel

MetricMeasuresGood
LCPLargest Contentful Paint — load speed< 2.5s
INPInteraction to Next Paint — responsiveness< 200ms
CLSCumulative Layout Shift — visual stability< 0.1
Mobile analogy: these are the web's version of frame-time / jank metrics. LCP ≈ time-to-first-meaningful-render; INP ≈ dropped frames on tap; CLS ≈ the layout jumping as content loads.

4Edge compute — run code, not just cache

Edge functions (Cloudflare Workers, Vercel Edge, Lambda@Edge) run at the CDN, milliseconds from the user. Perfect for auth checks, redirects, A/B splits, and personalization — without a trip to your origin region.

Where compute runs is now a design axis: edge (closest, limited) → regional server (Phase 5) → background worker (Phase 6). Push latency-sensitive work outward, heavy work inward.
