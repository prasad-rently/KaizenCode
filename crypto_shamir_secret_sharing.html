🧩 Shamir's Secret Sharing — Split a Secret

Split one secret into n pieces so that any k of them rebuild it — but k−1 reveal absolutely nothing. No piece is more important than another, and losing a few is fine. It's how you back up a key without trusting any single person or place.

A bonus module in the Cryptography topic. Unlike the encryption schemes elsewhere here, this isn't about hiding a message in transit — it's about distributing trust in a secret that already exists (a master key, a wallet seed, a root password).

1The idea

Analogy — the torn treasure map. Tear a map into pieces and hand them out so that any 3 of your 5 crew can reassemble it, but no 2 can. Nobody is a single point of failure: lose two pieces and the map still recovers; steal two and you learn nothing.

Formally, a (k, n) scheme produces n shares such that:

  • Any k shares reconstruct the secret exactly.
  • Any k−1 shares reveal nothing at all — not "almost the secret," but mathematically no information. This is perfect (information-theoretic) security.

2The trick: a polynomial through the secret

Recall from school geometry: 2 points define a line, 3 points define a parabola, and in general k points define a unique degree-(k−1) curve. Shamir turns that around:

  • Hide the secret as the curve's value at x = 0, i.e. f(0) = secret.
  • Pick the other k−1 coefficients at random, giving a random degree-(k−1) polynomial.
  • Each share is just a point on the curve: (1, f(1)), (2, f(2)), …
   With k = 3 (need 3 shares), the secret sits on a parabola:

     f(x)                       ● = a handed-out share (x, f(x))
      │        ●                 ★ = the secret f(0), never handed out
      │   ★  ●   ●
      │  ●        ●
      └────────────────── x
         0  1  2  3  4

   Any 3 points pin down exactly one parabola → recover f(0).
   Only 2 points? Infinitely many parabolas pass through them,
   each with a different f(0). The secret is undetermined.

That last line is the magic: with fewer than k shares, every possible secret is equally consistent with what you hold — so you've learned nothing.

3See it: split 3-of-5, then reconstruct

Below, the secret 1234 is split into 5 shares with threshold 3. Any 3 shares rebuild it via Lagrange interpolation at x = 0; 2 shares give garbage. (All arithmetic is over a prime field so shares leak no size information.) Run it:

JavaScript · runnable
// Shamir's Secret Sharing over a prime field
const P = 2147483647n;                 // 2³¹ − 1, a prime
const mod = a => ((a % P) + P) % P;
const egcd = (a, b) => { let or = a, r = b, os = 1n, s = 0n; while (r) { const q = or / r; [or, r] = [r, or - q * r]; [os, s] = [s, os - q * s]; } return [or, os]; };
const inv = a => mod(egcd(mod(a), P)[1]);
const rand = () => { const u = new Uint32Array(1); crypto.getRandomValues(u); return mod(BigInt(u[0])); };

const secret = 1234n, k = 3, n = 5;   // any 3 of 5 shares reconstruct
const coeffs = [secret];               // f(0) = secret
for (let i = 1; i < k; i++) coeffs.push(rand());  // k−1 random coefficients
const f = x => { let y = 0n, xp = 1n; for (const a of coeffs) { y = mod(y + a * xp); xp = mod(xp * x); } return y; };

const shares = [];
for (let x = 1n; x <= BigInt(n); x++) shares.push([x, f(x)]);
console.log('shares:', shares.map(s => `#${s[0]}=${s[1]}`).join('  '));

// reconstruct f(0) from ANY k shares via Lagrange interpolation
const recover = pts => { let s = 0n; for (let j = 0; j < pts.length; j++) { let num = 1n, den = 1n; for (let m = 0; m < pts.length; m++) if (m !== j) { num = mod(num * mod(-pts[m][0])); den = mod(den * mod(pts[j][0] - pts[m][0])); } s = mod(s + pts[j][1] * num % P * inv(den)); } return s; };

console.log('recover from #1,#3,#5 →', recover([shares[0], shares[2], shares[4]]).toString());
console.log('recover from #2,#4,#5 →', recover([shares[1], shares[3], shares[4]]).toString());
console.log('recover from only #1,#2 →', recover([shares[0], shares[1]]).toString(), '(garbage — below threshold)');

4Why a prime field

The math runs modulo a prime, not over plain integers. If shares were ordinary numbers, their size would leak hints about the secret. Working in a finite field makes every share look uniformly random in [0, p), which is what gives the "k−1 shares reveal nothing" guarantee its mathematical teeth. The secret and all shares must be smaller than the prime.

5Where it's used

  • Crypto-wallet recovery — split a seed phrase into shares kept in different places; any threshold restores the wallet (e.g. SLIP-39).
  • Root-key custody — HashiCorp Vault's unseal keys are Shamir shares; a quorum of operators must combine them to unseal.
  • Key escrow & break-glass — no single admin can access a master key alone; a threshold must cooperate.
  • HSMs & CAs — the "ceremony" to activate a root certificate key famously requires several key-holders together.
What it does and doesn't give you. Reconstruction brings the secret back together in one place — so it's for backup/custody, not for computing on a secret while keeping it split (that's threshold signatures / MPC). Plain Shamir also assumes honest shares: a malicious holder can submit a corrupted share and spoil the result. When that matters, use Verifiable Secret Sharing (VSS), which lets everyone check their share is valid.
Try it: (1) Split with k=3, then recombine shares 1,3,5 — exact secret. (2) Recombine only 1,2 — a random-looking number, and you can't tell which secret is real. (3) Set k=2 and see that now any 2 shares suffice — you tune the trade-off between resilience and how many holders must cooperate.
