🧰 The Cloud Toolbox

Every tool, service and utility a real application touches on its way from your laptop to a production cloud server — what each one is for, what it replaces, its alternatives, and whether you actually need it. Companion to AWS From Scratch, which explains the core services in depth.

1How to read this

Every tool below is tagged with how badly you need it. The trap for beginners is treating everything as required — real stacks start tiny and add tools when a specific pain appears:

  • MUST — production is irresponsible without it. There are 17 of these across the whole page — that's the entire mandatory surface.
  • RECOMMENDED — add in your first months; each one removes a whole class of 3am problems.
  • SITUATIONAL — solves a specific problem. Adopting it before you have that problem is how stacks become unmaintainable. The description says exactly when the moment arrives.

Filter the tables:

2Foundation — account, identity & secrets

Before an app can exist, someone has to answer: who are we in the cloud, who can touch what, and where do passwords live?

ToolWhat it is & its day-to-day jobNeed
AWS IAMThe permission system checking every API call. Daily: you assume roles, debug AccessDenied, and write least-privilege policies for services. Explained fully in From Scratch §5.Alternatives: none — it's mandatory. GCP IAM / Azure RBAC are the same idea elsewhere.MUST
AWS CLIEvery console button as a terminal command. Daily: aws logs tail, aws s3 sync, checking what's running without leaving your editor.Alternatives: the console (slower), SDKs (for code, not hands).MUST
Secrets Manager / SSM Parameter StoreThe vault for DB passwords, API keys and tokens. Apps fetch secrets at boot via IAM role — no secret ever sits in code, git, or a Slack message. Secrets Manager adds automatic rotation (~$0.40/secret/mo); Parameter Store is the free 90% version — start there. Daily: adding a key for a new integration, rotating one after an employee leaves.Alternatives: HashiCorp Vault (self-hosted, powerful, heavy), Doppler / Infisical (developer-friendly SaaS), 1Password for humans. A .env file on a server is the thing these all replace.MUST
IAM Identity Center (SSO) + AWS OrganizationsOne login for many AWS accounts, with dev/staging/prod as separate accounts so a dev mistake can't touch prod. Daily: aws sso login in the morning, switching roles between accounts.Alternatives: Okta/Entra as the IdP in front. Solo learners: one account + IAM user is fine.REC
ACM (Certificate Manager)Free, auto-renewing TLS certificates for your load balancer and CDN. Set once per domain, then forget certificates exist — which is the entire point.Alternatives: Let's Encrypt + certbot (when you manage your own web server), Cloudflare's edge certs.REC

3Shipping — source → build → deploy

The pipeline: code leaves a laptop, becomes an artifact, and lands in production without a human copying files. This whole stage is The Forge's territory.

ToolWhat it is & its day-to-day jobNeed
Git + GitHubSource of truth. Everything downstream is triggered by a push or merge. Daily: PRs, reviews, the merge button as your deploy button.Alternatives: GitLab (integrated CI), Bitbucket. AWS CodeCommit is deprecated — don't start there.MUST
GitHub Actions (CI/CD)Robot that runs on every push: tests → build → docker image → deploy. The YAML file in .github/workflows/ is your release process. Daily: watching a deploy go green, re-running a flaky job, adding a step.Alternatives: GitLab CI, CircleCI, Buildkite; AWS-native CodePipeline/CodeBuild (clunkier, IAM-native); Jenkins (self-hosted, legacy gravity).MUST
DockerPackages your app + runtime + OS deps into one image that runs identically on your laptop and in prod — ends "works on my machine" (Phase 4). Daily: docker build, docker compose up for local dev, writing the occasional Dockerfile.Alternatives: Podman (drop-in), Nixpacks/buildpacks (Dockerfile-free builds). Running Node directly on EC2 is the thing it replaces.MUST
ECR (Elastic Container Registry)Private shelf where built images live, versioned by tag; ECS/EKS pull from it. Daily: mostly invisible — CI pushes, compute pulls; you visit to check tags or clean old images with a lifecycle rule.Alternatives: Docker Hub, GitHub Container Registry (ghcr.io) — fine, but cross-cloud pulls cost latency and egress.MUST
Terraform (IaC)Your whole infrastructure — VPC, ECS, RDS, DNS — declared in files, reviewed in PRs, applied with one command. The cure for "we're afraid to touch the console" (The Blueprint). Daily: terraform plan on infra PRs, occasional state surgery.Alternatives: OpenTofu (open-source fork), AWS CDK (infra in TypeScript), Pulumi (any language), CloudFormation (raw YAML underneath). Any is fine; pick one and never click-create prod resources again.REC
Convox / Heroku-style PaaS on AWSAn opinionated platform that runs in your AWS account and hides the assembly: convox deploy creates the load balancer, ECS services, scaling, logs and secrets wiring for you — Heroku ergonomics, AWS bill. Daily (if adopted): it is your deploy tool, dashboard and log viewer.Alternatives on-AWS: Elastic Beanstalk, App Runner, Copilot CLI, SST (serverless apps), Flightcontrol, Porter. Off-AWS: Heroku, Render, Fly.io, Railway — genuinely great until traffic/cost/compliance pull you onto raw AWS. When: small team, no infra engineer, standard web app. When not: you need custom networking or you've outgrown the abstraction — the classic arc is PaaS → raw AWS + Terraform around year 2–3.SIT
Feature flags (LaunchDarkly / Unleash / Flagsmith)Turn features on/off per user at runtime, separating deploy from release. Daily: wrapping risky code in a flag, one-click kill switch when it misbehaves.Alternatives: a flags table in your own DB (fine at first), AWS AppConfig. When: the first time you wish you could undo a release without redeploying.SIT

4Running — compute, network & delivery

Where the app actually executes, and how a user's packet finds it. (Each core service explained from zero in From Scratch.)

ToolWhat it is & its day-to-day jobNeed
Compute: ECS + Fargate (or EC2 / Lambda / EKS)Something must run the container. The pragmatic default: ECS on Fargate — you say "run 3 copies of this image", AWS finds the machines; no servers to patch. Daily: watching a deploy roll, scaling a service, reading a stopped-task's reason.Alternatives: EC2 (cheapest, most control, most toil — the learning box), Lambda (event-driven/spiky), EKS/Kubernetes (when you have a platform team, not before), Beanstalk/App Runner (training wheels). One of these is MUST; Fargate is the default answer.MUST
VPCThe private network everything sits in: public subnets for the load balancer, private for app + DB. Set up once (in Terraform), then it's invisible until a connectivity bug drags you back in.Alternatives: none — only how much you customize the default.MUST
ALB (Application Load Balancer)The front door: terminates HTTPS (with the ACM cert), health-checks instances, spreads traffic, enables zero-downtime deploys. ~$16/mo+. Daily: invisible; you check its target-group health when something's down.Alternatives: API Gateway (for Lambda), nginx/Caddy on one box (single-server stage), Cloudflare in front.MUST
Route 53DNS — points yourapp.com at the ALB/CDN. Touched a few times a year; critical every second.Alternatives: Cloudflare DNS (free, fast, pairs with its CDN), your registrar's DNS (fine for simple setups).MUST
CloudFront (CDN)400+ edge caches serving your static assets (and cacheable API responses) from near the user, cutting latency and your servers' load; also your DDoS buffer and free TLS at the edge. Daily: invalidating a cache after a bad asset ships.Alternatives: Cloudflare (free tier is genuinely good — common to run Cloudflare in front of AWS), Fastly. When MUST: any real static frontend or global users.REC
Auto ScalingRules that add/remove app copies with load ("keep CPU ≈ 60%"). Set once; it quietly pays for itself every traffic spike and every quiet night.Alternatives: overprovisioning (money) or manual scaling (sleep).REC
API GatewayManaged front door for Lambda-based APIs: routing, auth (JWT/Cognito), rate limiting, per-request pricing. When: serverless architecture. Container apps behind an ALB don't need it.Alternatives: ALB (cheaper at volume), Kong/Tyk (self-hosted gateways — The Gateway).SIT

5Data — databases, caches, queues & search

ToolWhat it is & its day-to-day jobNeed
RDS Postgres (or Aurora)The system of record. Managed backups, patching, Multi-AZ failover, read replicas (From Scratch §9). Daily: slow-query hunting, the occasional migration, checking replica lag. Aurora = AWS's re-engineered Postgres/MySQL — faster and pricier; start plain RDS.Alternatives: Neon / Supabase / PlanetScale (serverless DBaaS, lovely DX), MongoDB Atlas (if document-shaped), self-hosted on EC2 (don't, until you have a reason and a DBA).MUST
S3Files: uploads, builds, backups, logs, data lake — everything that isn't a database row ends up here. Daily: presigning upload URLs, lifecycle rules aging logs to Glacier.Alternatives: Cloudflare R2 (zero egress fees — interesting for serving big files), GCS/Azure Blob elsewhere.MUST
ElastiCache RedisIn-memory store for the hot path: sessions, rate-limit counters, cached queries, pub/sub, job queues (BullMQ). Usually the first infra added after the DB, the moment something is slow or needs sharing across instances.Alternatives: Upstash (serverless Redis, per-request pricing), Valkey (open-source fork), Memcached (pure cache). In-process memory works until instance #2 exists.REC
SQS (+ SNS / EventBridge)Queues — the shock absorber. Web tier enqueues slow work (emails, resizing, webhooks); workers chew through it at their own pace; traffic spikes become queue depth instead of timeouts. SNS fans one event to many listeners; EventBridge routes events by pattern and runs cron. Daily: watching queue depth, draining a dead-letter queue after a bad deploy.Alternatives: Redis + BullMQ (simpler start), RabbitMQ, Kafka/MSK (event streaming at scale — Phase 6, The Streaming Mind).REC
DynamoDBServerless NoSQL: single-digit-ms key-value lookups at any scale, zero maintenance. When: Lambda-native stacks, or a table with brutal scale and simple access patterns (sessions, carts, device state). Modeling is unforgiving — design access patterns first.Alternatives: Redis (hot ephemeral), Postgres JSONB (honestly fine for most "NoSQL" needs), MongoDB Atlas.SIT
OpenSearch / a search engineFull-text search, fuzzy matching, relevance ranking, log analytics — what LIKE '%…%' can't do. When: search becomes a feature users notice.Alternatives: Postgres full-text search (surprisingly far), Algolia (best DX, $$), Meilisearch/Typesense (light, self-hostable), Elastic Cloud.SIT
Data warehouse (Redshift / BigQuery / Snowflake)Analytics home — dashboards and "how many users did X" queries run here, not on your production DB. When: the first time an analyst's query slows production (The Ledger).Alternatives: a read replica + Metabase is the honest first step; DuckDB over S3 for small scale.SIT

6Watching — observability & on-call

Production without observability is a car with the windshield painted black (The Watchtower).

ToolWhat it is & its day-to-day jobNeed
CloudWatch (logs + metrics + alarms)Built-in eyes: every service emits metrics here, containers ship logs here, alarms page you. Daily: aws logs tail --follow during a deploy, checking a latency graph, tuning an alarm threshold. It's nobody's favorite UI, but it's already there and already integrated.Alternatives/upgrades: Datadog (best-in-class, watch the bill), Grafana + Loki + Prometheus (open-source stack), New Relic, Better Stack.MUST
Sentry (error tracking)Catches every unhandled exception with stack trace, user, release, and breadcrumbs; groups duplicates; tells you which deploy broke it. The single highest value-per-dollar tool on this page. Daily: triaging the morning's new issues, linking one to a PR.Alternatives: Rollbar, Bugsnag, Honeybadger, self-hosted GlitchTip. Grepping logs is the thing it replaces.MUST
Uptime monitoring (UptimeRobot / Better Stack / Pingdom)Something outside your cloud that curls your health endpoint every minute and screams if it fails — because when your region is down, so is your monitoring. Five minutes to set up.Alternatives: CloudWatch Synthetics (inside AWS — misses whole-account outages), cron + curl somewhere else.REC
Tracing (X-Ray / OpenTelemetry + Jaeger/Honeycomb)Follows one request across services showing where the milliseconds went. When: more than ~3 services and "which hop is slow?" stops being answerable from logs.Standardize on OpenTelemetry instrumentation regardless of backend — it keeps you portable.SIT
PagerDuty / Opsgenie (on-call)Turns alerts into an escalating phone call to whoever's on rotation, with schedules and handoffs. When: more than ~3 engineers share production responsibility.Alternatives: CloudWatch → SNS → phone/Slack (fine for a solo app), Grafana OnCall (free).SIT
Status page (Atlassian Statuspage / Instatus / Better Stack)Public "is it down?" page that cuts support tickets in half during incidents. When: paying customers exist.SIT

7Guarding — security, audit & cost

ToolWhat it is & its day-to-day jobNeed
CloudTrailThe account's flight recorder: every API call — who, what, when, from where. Turn on once, forget until an incident or audit, then thank yourself. Effectively free.Alternatives: none. This is the audit trail.MUST
Budgets + Cost ExplorerThe $-alarm and the where-did-it-go dashboard. Daily-ish: a Friday five-minute glance at the daily-spend graph catches every mistake before it compounds (The Wallet).Alternatives/upgrades: Vantage, CloudZero (multi-account cost SaaS) — later, at org scale.MUST
Dependabot / Renovate + a scanner (Snyk/Trivy)Robots that PR your dependency updates and flag known-vulnerable packages and images. Supply-chain security on autopilot (The Forge).Dependabot is free in GitHub — no excuse. Trivy scans Docker images in CI.REC
GuardDutyAWS's threat detector: watches CloudTrail/VPC/DNS telemetry for cryptominers, leaked-key usage, port scans. One click on, ~dollars/month, zero maintenance.Alternatives: none as cheap for the coverage. SIEMs come much later.REC
WAF (+ Shield)HTTP firewall on your ALB/CloudFront: blocks SQL-injection patterns, bad bots, per-IP floods. When: public forms, login endpoints, or the first scraping/DDoS scare. Shield Standard (DDoS) is free and automatic.Alternatives: Cloudflare WAF (often simpler and cheaper in front of everything).SIT
Security Hub / Inspector / ConfigContinuous compliance: benchmark checks ("S3 buckets public?", "MFA everywhere?"), resource-change history, finding aggregation. When: SOC 2 / ISO audits appear on the horizon — auditors ask for exactly these reports.SIT

8At your desk — daily developer utilities

ToolWhat it is & its day-to-day jobNeed
SSM Session ManagerShell into any instance/container through IAM — no SSH keys, no port 22 open, every session audited. The modern replacement for the .pem file workflow you learned with.Alternatives: SSH + bastion host (the classic), Tailscale SSH, EC2 Instance Connect.REC
aws-vault / Granted / LeappKeeps AWS credentials encrypted in your OS keychain and issues short-lived session tokens per shell — instead of plaintext keys in ~/.aws/credentials waiting to be exfiltrated. Daily: assume prod-readonly.With SSO configured, aws sso login covers much of this natively.REC
Docker Compose (local stack)One docker compose up starts your app + Postgres + Redis on your laptop, mirroring prod's shape. The daily inner loop.Alternatives: running services natively via brew (drift city), devcontainers, Tilt (at k8s scale).REC
LocalStackA fake AWS (S3, SQS, Lambda, DynamoDB…) running in a container so integration tests don't need a real account. When: your test suite starts touching AWS APIs.Alternatives: a dedicated sandbox AWS account (more real, more risk), MinIO for just-S3.SIT
k6 / Artillery (load testing)Scripted fake traffic to find your breaking point before launch day does. When: before any launch you're nervous about (The Edge).SIT
ngrok / Cloudflare TunnelPublic URL for your laptop — testing webhooks (Stripe, GitHub) against local code without deploying.SIT
infracostPrints the monthly cost delta of a Terraform PR right in the review — "this change adds $340/mo" as a review comment. When: IaC is established and bills matter.SIT

9The minimal viable production stack

Strip away everything situational and this is the honest day-one shopping list — 13 pieces run a real production web app responsibly:

# identity & safety
IAM + CLI            → who may do what
Parameter Store      → where secrets live
CloudTrail + Budgets → audit log + $-alarm

# shipping
GitHub + Actions     → source + robot that deploys
Docker + ECR         → the artifact + its shelf

# running
VPC + ALB (+ACM)     → network + HTTPS front door
ECS on Fargate       → runs the containers
Route 53             → the name

# state
RDS Postgres         → the data
S3                   → the files

# eyes
CloudWatch + Sentry  → metrics/logs/alarms + every exception

Everything else on this page earns its place later, when its specific pain shows up: Redis when something's slow, SQS when work goes async, CloudFront when users are far away, WAF after the first bot swarm, PagerDuty when the team shares the pager, Terraform the moment two environments drift.

The anti-pattern is adopting the whole page on day one. Every tool is surface area: something to configure, secure, update and pay for. Great stacks are grown, not installed.

Build a feature locallyDocker Compose (app+DB+Redis), ngrok if webhooks involved
Ship itgit push → PR review → merge → GitHub Actions: tests, docker build, push to ECR, ECS rolling deploy behind the ALB → watch Sentry's release tab for 10 min
Add a third-party API keySecrets/Parameter Store (never .env, never Slack) → app reads it via IAM role at boot
